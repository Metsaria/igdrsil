# Projects and Tasks

Igdrsil keeps work deliberately simple.

- **Project**: one matter, client engagement, or work area.
- **Task**: one concrete piece of work inside a Project.

There are no nested folders and no hidden workspace hierarchy.

## Create a Project

1. Click **New Project**.
2. Enter a clear Project name.
3. Add a short description if useful.
4. Click **Create**.

Use names your colleagues will recognise later, for example "Acme contract review" or "Quarterly policy update".

## Create a Task

1. Open the Project.
2. Click **New Task**.
3. Name the Task after the specific job, for example "Review draft contract".
4. Choose the agents that should be available in the Task.
5. Click **Create**.

Each Task has its own chat, uploaded files, agent outputs, and history.

## Invite colleagues

Only invited Project members can see a Project. To add someone, open the Project participants panel and invite the colleague. Access is scoped per Project; there is no default setting that exposes every Project to the whole firm.

## Archive finished work

When the matter or work area is finished, archive the Project. Archiving removes it from the daily list but keeps the history on the firm's server.
