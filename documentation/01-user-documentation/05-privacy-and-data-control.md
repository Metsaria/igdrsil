# Privacy and Data Control

Igdrsil is built around one rule: customer data stays inside the customer's infrastructure.

## Where data lives

Your firm's data lives on the Igdrsil server your firm controls. That includes:

- chat messages;
- uploaded files;
- Project and Task history;
- document search indexes;
- agent reports;
- technical logs written by the server.

## What leaves the server

In normal operation, no firm data leaves the server. AI processing runs locally. The vendor cannot read your documents, messages, Corpus Sources, or agent reports.

Outbound network access is limited to administrator-controlled actions such as updates or explicitly enabled connectors. Those flows must not upload firm content.

## No phone-home analytics

Igdrsil does not send usage analytics, document samples, prompts, task names, logs, or error reports back to the vendor.

## Support model

Because the vendor has no back door into the server, support starts with your administrator. If vendor support is needed, your firm decides what evidence to share.

## Deleting work

Project deletion, user deactivation, and data-subject erasure are administrator-controlled actions. Ask your administrator if data needs to be archived, exported, or deleted.
